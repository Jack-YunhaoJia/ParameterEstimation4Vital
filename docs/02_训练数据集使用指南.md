# Vital 合成器参数估计训练数据集使用指南

> 本文档面向负责模型训练的协作者，介绍数据集的格式、内容和使用方法。

## 1. 数据集概述

本数据集包含 94,586 个（合成器参数, 音频 embedding）配对样本，用于训练从音频反推 Vital 合成器参数的回归模型。数据已按 80/10/10 划分为训练集、验证集和测试集。

### 核心特点
- **参数维度**：45 维连续/离散混合参数向量
- **音频表示**：1024 维 MuQ embedding（预训练音乐理解模型）
- **多条件**：每个预设在 6 种 MIDI 条件下渲染（3 音高 × 2 力度）
- **存储格式**：HDF5，按 train/val/test 划分，支持高效随机访问

### 数据集规模

| 划分 | 样本数 | 说明 |
|------|--------|------|
| train | 75,668 | 80% |
| val | 9,458 | 10% |
| test | 9,460 | 10% |
| 总计 | 94,586 | 从 102,042 个渲染中过滤 7,456 个无效样本 |

### 3.2 参数归一化

数据集中的参数值为**原始物理值**（非归一化）。训练时建议归一化到 [0, 1]：

```python
with h5py.File("production_dataset.h5", "r") as f:
    param_ranges = f["metadata"]["param_ranges"][:]  # (45, 2)

params_normalized = (params - param_ranges[:, 0]) / (param_ranges[:, 1] - param_ranges[:, 0])
```

### 3.3 参数类型处理

训练时需注意不同参数类型的处理：

- **连续参数**（36 个）：标准回归目标，使用 MSE 或 L1 损失
- **离散参数**（2 个）：`filter_1_model`（6 类）和 `filter_1_style`（4 类），建议使用分类损失或量化后的回归
- **二值参数**（9 个）：效果器开关，建议使用 BCE 损失，推理时以 0.5 为阈值

```python
# 推荐的混合损失函数
continuous_indices = [i for i in range(45) if i not in discrete_indices + switch_indices]
discrete_indices = [10, 11]  # filter_model, filter_style
switch_indices = list(range(16, 25))  # 9 个效果器开关

loss = (
    mse_loss(pred[:, continuous_indices], target[:, continuous_indices])
    + ce_loss(pred_model_logits, target[:, 10].long())
    + ce_loss(pred_style_logits, target[:, 11].long())
    + bce_loss(pred[:, switch_indices], target[:, switch_indices])
)
```

## 4. Embedding 说明

### 4.1 MuQ 模型

音频 embedding 由 MuQ 模型提取（`OpenMuQ/MuQ-large-msd-iter`）：
- 输入：16kHz 单声道音频
- 输出：1024 维 embedding 向量
- 特点：在大规模音乐数据上预训练，对音色、节奏、和声等音乐特征有良好表示

### 4.2 Embedding 使用

训练时，embedding 作为模型输入（音频的紧凑表示），参数向量作为预测目标：

```
模型结构: MuQ Embedding (1024) → MLP/Transformer → 参数向量 (45)
```

## 5. 多条件数据的使用

### 5.1 条件信息

每个样本附带 MIDI 条件信息，通过 `midi_notes` 和 `midi_velocities` 字段获取：

| MIDI Note | 音高 | MIDI Velocity | 力度 |
|-----------|------|---------------|------|
| 48 | C3 | 80 | 中等 |
| 48 | C3 | 120 | 强 |
| 60 | C4 | 80 | 中等 |
| 60 | C4 | 120 | 强 |
| 72 | C5 | 80 | 中等 |
| 72 | C5 | 120 | 强 |

### 5.2 使用策略

**策略 A：条件无关训练**
将所有 6 个条件的样本混合训练，不区分条件。模型需要从不同条件的音频中学习到相同的参数映射。

```python
# 直接使用全部训练数据
train_loader = DataLoader(dataset, batch_size=64, shuffle=True)
```

**策略 B：条件感知训练**
将条件编码为额外输入特征，让模型学习条件与参数的交互关系。

```python
# 将 MIDI 条件编码为额外输入
note_embed = note_embedding(midi_notes)      # (N, d)
vel_embed = velocity_embedding(midi_vels)    # (N, d)
model_input = torch.cat([embeddings, note_embed, vel_embed], dim=-1)
```

**策略 C：单条件训练**
只使用 C4_v80（中音区中等力度）训练，作为基线实验。

```python
mask = (midi_notes == 60) & (midi_velocities == 80)
train_params = params[mask]
train_embeds = embeddings[mask]
```

## 6. 数据质量说明

### 6.1 已过滤的样本

以下类型的样本在生产过程中已被过滤，不会出现在数据集中：
- **静音样本**：7,198 个（RMS < -60 dBFS，通常由低 osc_level + 长 attack 组合导致）
- **削波样本**：258 个（削波比例 > 10%）
- 总过滤率：7.3%

### 6.2 音频预处理标准

所有保留的音频样本均经过：
- DC 偏移消除
- 峰值归一化至 -1.0 dBFS
- 重采样至 16kHz
- 尾部静音裁剪（最短 0.5 秒）

### 6.3 近似重复

质量验证阶段检测到 12,024 对近似重复样本（embedding 余弦相似度 > 0.999）。这些样本已标记但未移除，因为它们对应不同的参数组合，只是在 embedding 空间中表现相似。

## 7. 快速开始示例

```python
import h5py
import numpy as np
import torch
from torch.utils.data import Dataset, DataLoader

class VitalSynthDataset(Dataset):
    def __init__(self, h5_path, split="train", condition_note=None):
        with h5py.File(h5_path, "r") as f:
            grp = f[split]
            params = grp["params"][:]
            embeds = grp["embeddings"][:]
            notes = grp["midi_notes"][:]
            vels = grp["midi_velocities"][:]
            ranges = f["metadata"]["param_ranges"][:]

        # 可选：按条件过滤
        if condition_note is not None:
            mask = notes == condition_note
            params, embeds, notes, vels = params[mask], embeds[mask], notes[mask], vels[mask]

        # 归一化参数到 [0, 1]
        self.params = (params - ranges[:, 0]) / (ranges[:, 1] - ranges[:, 0])
        self.embeds = embeds
        self.notes = notes
        self.vels = vels

    def __len__(self):
        return len(self.params)

    def __getitem__(self, idx):
        return (
            torch.tensor(self.embeds[idx], dtype=torch.float32),
            torch.tensor(self.params[idx], dtype=torch.float32),
        )

# 使用
train_set = VitalSynthDataset("production_dataset.h5", split="train")
val_set = VitalSynthDataset("production_dataset.h5", split="val")
test_set = VitalSynthDataset("production_dataset.h5", split="test")

train_loader = DataLoader(train_set, batch_size=64, shuffle=True)

for embeddings_batch, params_batch in train_loader:
    # embeddings_batch: (64, 1024)
    # params_batch: (64, 45)
    predictions = model(embeddings_batch)
    loss = criterion(predictions, params_batch)
    ...
```
