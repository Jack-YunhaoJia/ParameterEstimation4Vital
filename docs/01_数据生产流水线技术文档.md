# Vital 合成器参数-音频数据集生产流水线技术文档

## 1. 项目概述

本项目为 Vital 合成器参数估计研究构建大规模训练数据集。最终生成了 94,586 个高质量的（参数向量, 音频 embedding）配对样本，用于训练从音频反推合成器参数的深度学习模型。

### 1.1 核心思路

合成器参数估计的训练数据需要满足：
- **参数空间覆盖率高**：45 维参数空间需要均匀覆盖，避免采样偏差
- **音频质量可控**：排除静音、削波等无效样本
- **多条件渲染**：同一预设在不同音高/力度下渲染，增加数据多样性
- **标准化预处理**：所有音频经过统一的预处理流水线，确保一致性

### 1.2 技术栈

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| 合成器 | Vital VST3 | 开源 wavetable 合成器，通过 pedalboard 加载 |
| VST 宿主 | pedalboard 0.9.22 | Spotify 开源的 Python VST 宿主库 |
| 音频 embedding | MuQ (OpenMuQ/MuQ-large-msd-iter) | 音乐理解模型，输出 1024 维 embedding |
| 采样策略 | scipy LHS | 拉丁超立方采样 + 分层效果器开关 |
| 存储格式 | HDF5 | 高效的大规模数值数据存储，按 train/val/test 划分 |
| 硬件 | macOS M4 Pro | 12 核 CPU + MPS GPU 加速 embedding 提取 |

## 2. 流水线架构

### 2.1 整体流程

```
SmartSampler (LHS采样)
    ↓ 45维参数向量 × 17,007
PresetGenerator (生成 .vital 预设文件)
    ↓ .vital JSON 文件
AudioRenderer (pedalboard + Vital VST3)
    ↓ 原始 WAV (44.1kHz, 32-bit float)
MultiConditionRenderer (6条件渲染)
    ↓ 每预设 6 个 WAV 文件 → 共 102,042 个
AudioPreprocessor (预处理流水线)
    ↓ 标准化 WAV (16kHz, 峰值归一化)
    ↓ 过滤静音/削波 → 7,456 个被过滤
EmbeddingExtractor (MuQ 模型, MPS GPU)
    ↓ 1024 维 embedding 向量
QualityValidator (质量验证)
    ↓ 近似重复检测 (12,024 对)
HDF5 Writer (最终输出)
    ↓ production_dataset.h5 (94,586 样本)
```

## 3. 参数空间定义

### 3.1 CORE_PARAMS（45 维）

我们选择了 45 个对音色影响最显著的 Vital 参数作为核心参数空间：

**振荡器参数（6 个）**：
- `osc_1_level` [0, 1]：振荡器 1 音量
- `osc_1_transpose` [-48, 48]：半音移调
- `osc_1_tune` [-1, 1]：精细调音
- `osc_1_wave_frame` [0, 256]：波表帧位置
- `osc_1_unison_voices` [1, 16]：齐奏声部数
- `osc_1_unison_detune` [0, 100]：齐奏失谐量

**滤波器参数（6 个）**：
- `filter_1_cutoff` [8, 136]：截止频率（MIDI note）
- `filter_1_resonance` [0, 1]：共振
- `filter_1_drive` [0, 1]：驱动
- `filter_1_mix` [0, 1]：干湿比
- `filter_1_model` [0, 5]：滤波器类型（离散）
- `filter_1_style` [0, 3]：滤波器风格（离散）

**包络参数（4 个）**：
- `env_1_attack` [0, 4]：起音时间（秒）
- `env_1_decay` [0, 4]：衰减时间（秒）
- `env_1_sustain` [0, 1]：延持电平
- `env_1_release` [0, 4]：释放时间（秒）

**效果器开关（9 个，二值）**：
- `chorus_on`, `compressor_on`, `delay_on`, `distortion_on`, `eq_on`, `flanger_on`, `phaser_on`, `reverb_on`, `filter_fx_on`

**效果器混合参数（9 个）**：
- `chorus_dry_wet`, `delay_dry_wet`, `flanger_dry_wet`, `phaser_dry_wet`, `reverb_dry_wet`, `distortion_mix`, `compressor_mix`, `filter_fx_mix`, `reverb_decay_time`

**其他效果器参数（11 个）**：
- `distortion_drive`, `delay_frequency`, `chorus_frequency`, `chorus_feedback`, `flanger_frequency`, `phaser_frequency`, `reverb_size`, `delay_feedback`, `compressor_attack`, `compressor_release`, `eq_low_gain`

### 3.2 采样策略

采用 **LHS + 分层效果器开关** 策略（`lhs_stratified`）：

1. **连续参数**：使用拉丁超立方采样（Latin Hypercube Sampling），确保每个参数维度的边际分布均匀
2. **离散参数**：`filter_1_model` 取整到 {0,1,2,3,4,5}，`filter_1_style` 取整到 {0,1,2,3}
3. **效果器开关**：按活跃效果器数量（0-9）分层采样，每层样本数与组合数 C(9,k) 成正比

这种策略相比朴素均匀采样，能显著提升参数空间覆盖率（通过 KS 检验验证）。

## 4. 多条件渲染

每个预设在 6 种 MIDI 条件下渲染：

| 条件标签 | MIDI Note | 力度 | 时长 | 说明 |
|----------|-----------|------|------|------|
| C3_v80   | 48 (C3)   | 80   | 2.0s | 低音区，中等力度 |
| C3_v120  | 48 (C3)   | 120  | 2.0s | 低音区，强力度 |
| C4_v80   | 60 (C4)   | 80   | 2.0s | 中音区，中等力度 |
| C4_v120  | 60 (C4)   | 120  | 2.0s | 中音区，强力度 |
| C5_v80   | 72 (C5)   | 80   | 2.0s | 高音区，中等力度 |
| C5_v120  | 72 (C5)   | 120  | 2.0s | 高音区，强力度 |

设计理由：
- **3 个八度**（C3/C4/C5）覆盖合成器常用音域，捕捉音色随音高的变化
- **2 种力度**（80/120）捕捉力度响应特性
- **2 秒时长**足以包含完整的 ADSR 包络周期

## 5. 音频预处理流水线

渲染后的原始音频（44.1kHz, 32-bit float）经过以下标准化处理：

```
原始音频 (44.1kHz)
    ↓
① 静音检测: RMS < -60 dBFS → 过滤 (7,198 个)
    ↓
② 削波检测: |sample| > 0.99 的比例 > 10% → 过滤 (258 个)
    ↓
③ DC 偏移消除: audio -= mean(audio)
    ↓
④ 峰值归一化: 归一化至 -1.0 dBFS
    ↓
⑤ 重采样: 44.1kHz → 16kHz (scipy resample_poly, 抗混叠)
    ↓
⑥ 尾部静音裁剪: 移除尾部 RMS < -50 dBFS 的静音段
    ↓
标准化音频 (16kHz, 最短 0.5s)
```

### 预处理配置

| 参数 | 值 | 说明 |
|------|-----|------|
| silence_threshold_db | -60.0 | 静音检测阈值 |
| clipping_threshold | 0.99 | 削波检测绝对值阈值 |
| clipping_ratio_limit | 0.10 | 削波比例上限 |
| target_peak_db | -1.0 | 峰值归一化目标 |
| target_sample_rate | 16000 | MuQ 模型要求的采样率 |
| tail_silence_threshold_db | -50.0 | 尾部静音裁剪阈值 |
| min_duration_sec | 0.5 | 最小有效音频时长 |

## 6. 质量验证

### 6.1 样本级验证

每个音频样本经过以下质量检查：
- **静音检测**：RMS 能量低于阈值的样本被过滤（占 7.1%）
- **削波检测**：削波比例超过阈值的样本被过滤（占 0.3%）

### 6.2 数据集级验证

- **近似重复检测**：基于 embedding 余弦相似度（阈值 0.999），检测到 12,024 对近似重复样本（已标记，未移除）
- **Embedding 多样性**：PCA top-10 累积方差 = 53.86%，表明 embedding 空间具有良好的多样性，无维度坍缩

## 7. 并行生产架构

```
主进程 (ParallelProducer)
    ├── SmartSampler: 生成 17,007 个参数向量
    ├── PresetGenerator: 生成 .vital 预设文件
    ├── 11 个渲染 Worker (ProcessPoolExecutor)
    │   ├── Worker 0: AudioRenderer + MultiConditionRenderer
    │   ├── Worker 1: AudioRenderer + MultiConditionRenderer
    │   ├── ...
    │   └── Worker 10: AudioRenderer + MultiConditionRenderer
    ├── AudioPreprocessor: 预处理所有渲染音频
    ├── EmbeddingExtractor (MPS GPU): 批量提取 embedding
    ├── QualityValidator: 质量验证 + 近似重复检测
    └── HDF5 Writer: 写入最终数据集 (80/10/10 划分)
```

关键设计：
- 每个 Worker 独立加载 Vital VST3 实例，避免插件状态冲突
- 每 100 个预设保存一次 checkpoint，支持断点恢复
- Embedding 提取使用 MPS GPU 加速，batch_size=32

## 9. 运行方式

```bash
# 完整生产运行
python3 scripts/run_production.py --output-dir experiments/production_run -y

# 指定目标样本数
python3 scripts/run_production.py --output-dir experiments/production_run --target-samples 10000 -y

# 断点恢复
python3 scripts/run_production.py --output-dir experiments/production_run --resume

# 自定义配置
python3 scripts/run_production.py --output-dir experiments/production_run --config configs/production.yaml --n-workers 8 --device mps -y
```
